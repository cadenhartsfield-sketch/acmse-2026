# ACMSE 2026 Programming Contest – Image Segmentation

## Overview

This project implements a **grayscale image segmentation** algorithm that partitions every pixel in an image into distinct regions (segments) based on local intensity similarity and 4-directional adjacency, as specified by the ACMSE 2026 contest rules.

## Algorithm

### Rule

Two pixels belong to the same segment if and only if:
1. They are **4-directionally adjacent** (up / down / left / right — no diagonals).
2. Their absolute intensity difference is **≤ 15**.

This defines an equivalence relation over pixels, and the segments are exactly the connected components of the graph where edges connect pixels satisfying both conditions.

### Implementation: BFS Connected Components

The algorithm performs a standard **raster-scan BFS flood-fill**:

```
for every pixel (r, c) in row-major order:
    if (r, c) is unvisited:
        assign a new segment ID
        BFS from (r, c), expanding to all 4-directional neighbours
        where |intensity(current) − intensity(neighbour)| ≤ 15
```

**Why BFS and not DFS?**  
BFS processes pixels level-by-level, keeping the queue shallow relative to the image width. DFS risks hitting Python's recursion limit (~1 000) on large contiguous regions.

**Why not Union-Find?**  
Union-Find would also be correct, but BFS is simpler to read and audit, and its runtime is identical: **O(H × W)** — every pixel is enqueued and dequeued at most once.

### Segment ID Encoding

With threshold = 15 on natural images, a 321 × 481 image can produce **up to ~37 000 segments** — well above the 255-value limit of `uint8`. Masks are therefore saved as **16-bit PNGs** (`PIL` mode `I;16`), which supports up to 65 535 distinct IDs.

---

## Complexity

| Dimension | Value |
|-----------|-------|
| Time      | O(H × W) — each pixel processed exactly once |
| Space     | O(H × W) — labels array + BFS queue (worst case full image) |

On a 321 × 481 image, runtime is approximately **0.2 seconds per image** in pure Python.

## Usage

### 1 – Generate predictions for test set

```bash
python segment.py
# Output: predicted_masks/<filename>.png  for each test image
#         predicted_masks/summary.csv     segment counts
```

### 2 – Verify rule compliance

```bash
python scripts/verify.py
# Checks: all pixels assigned, no adjacent similar pair split across segments
```

### 3 – Visualize results

```bash
python scripts/visualize.py               # all 50 test images
python scripts/visualize.py --file 108082.png   # single image
# Output: visualizations/<filename>.png  (original | colorized segments)
```

---

## Correctness Guarantee

By construction, BFS flood-fill **cannot** produce a rule violation: a neighbour is added to the current segment only when its intensity difference from the current pixel is ≤ 15. This is verified automatically by `verify.py`, which confirmed **0 violations** across all 50 test masks.
